LOCOMO——长期对话数据集+评估框架

ACL 2024 论文《Evaluating Very Long-Term Conversational Memory of LLM Agents》阅读笔记。

会议 ACL 2024

TL;DR：提出长期对话数据集 LOCOMO，提出包含三类任务的评估标准，对三类 LLM 进行测试（基础 LLM、长上下文 LLM、RAG 增强 LLM）。

一、问题背景
#

1. 长上下文大语言模型（Long-context LLMs）
#

  • 扩展上下文窗口（一次性可输入的文本长）、优化注意力（滑动窗口、稀疏注意力、注意力池化）等方式，实现超长文本的语义理解。
  • Long-context 只是可以单次处理更长的上下文，并不具有长期记忆能力，即不重新输入上下文，就会全忘记。
  • 缺点：Long-context LLMs 上下文一旦长了，处理新问题需要前文 + 新输入，每一次调用会消耗大量 token，成本高。上下文问题是瓶颈

2. 检索增强生成大语言模型（RAG LLMs）
#

  • RAG 从外部知识库 / 对话历史中检索出关键信息作为上下文输入 LLM，既可以缓解 token 消耗成本问题，也避免了全量长文本输入下可能的干扰信息导致的幻觉问题。
  • 流程：建库（拆长文本为检索单元）——检索（匹配关键单元）——生成（关键信息 + 问题输入）。
  • 缺点：检索漏检风险、开放域知识问题表现差，检索单元质量依赖性强。
    • RAG-LLM 本质是检索外部私有信息（如本轮对话历史），而非检索通用知识，当问题需要外部知识时，回答准确率大幅下降。

3. 数据集评估的先天缺陷
#

  • 现有数据集篇幅短、会话少，单模态，无法支撑对超长期对话的评估。

二、前人研究
#

1. 长期对话中的推理
#

（1）检索历史对话并进行时序推理

AI 聊新内容前，先从之前的聊天记录翻出相关内容，按时间顺序梳理清楚。

（2）用“事件”搭建对话框架

梳理关键事件，按事件发展脉络聊。如把“想养柯基 -> 查柯基习性 -> 选狗粮 -> 准备狗窝”设为一串关联事件，AI 会跟着这个事件框架聊，不会聊到选狗粮时突然问“你为什么想养猫”。

局限：

  • 检索错误：AI 基于语义相似检索，但现实聊天可能有指代、模糊的情况，可能检索错误。
  • 检索不准：AI 能检索到一堆相关历史，但是分不清哪句是核心。比如“想养柯基，预算不多，还喜欢冻干狗粮”，这周你问“性价比高的推荐下”，AI 却盯着“喜欢冻干”推荐贵的冻干狗粮，完全忽略“预算不多”的核心信息。
  • 跨时间聊天：间隔不同的时间聊同一件事情，说法 / 关注点可能完全不同，但是 AI 无法做到。

2. 多模态对话
#

  • 基于图像的对话：基于图像，构建文本（图像问答等）。
  • 图像共享对话：基于文本，选择图像。

论文结合了上述两种方法，文本 -> 图像 -> 文本的逻辑。

3. LLM 生成数据
#

LLM 合成不同场景的对话基准，如评估日常社交聊天、多模态环境、符合任务设定的回应。

论文除了使用 LLM 生成，还基于人工验证与编辑。

三、核心贡献
#

1. LOCOMO 数据集构建
#

  1. 理论依据：真实人类对话，是多维度的融合（话题、语言风格、对话内容、共同记忆）。

    • 集体记忆：双方在交互过程中形成的共享记忆。“上次去的咖啡店”
    • 个人观点：说话者基于自身经历、认知形成的独立看法、观点和偏好。“三观 -> 具体事件的自己看法 -> 说话内容”
    • 外部影响：说话者所属环境等客观影响，是对话话题的重要来源。如换工作了、旅行了。时序事件图的构建
    • 说话者独特人格特质：说话者性格、行为方式、语言风格，决定对话的风格。“说话形式”

  2. 构建流程

流程：

另一方发送了新消息（Ongoing Session），Agent 首先通过一个检索器将长期记忆中与当前语义相关的记忆调出（长期记忆存储的是 Observation-Reflection 对，Observation 即事实细节，Reflection 即总结与推论）。如果新消息含有图片，则会调用 Image Reaction 进行语义解析。Short-term memory 的作用是临时缓存本轮会话。

Prediction 的输入包括记忆类、个人设定类、事件类。输出文本 Response 或者图片（Image Sharing），Image Sharing 作为 Prediction 的输入，用以判断是否需要发图。

  • 调用 LLM，基于人格设定 + 人生时间线 + 图像分享与反应（多模态）三大机制生成对话。对话具有人设一致、事件连贯、记忆闭环、多模态融合等特点。
    • 人物设定：MSC 数据集 + LLM 扩展（gpt-3.5-turbo），包含目标、过往经历、日常习惯、人际关系，以及个体的姓名、年龄和性别。
    • 时序事件图：基于人物设定，LLM 生成时间、因果关联的人生事件。
    • Agent 架构：
      • 反思与应答：回应涉及短期与长期记忆。会话结束后，基于当前会话和之前的摘要生成一个新的摘要，存入短期记忆；单轮对话转换为 observation，存入长期记忆。
      • 图片反应和分享：
        • 图片分享：决定是否生成图片 -> 调用模型生成描述 -> 描述转换为关键词 -> 基于关键词网络搜索查找。
  • 人工标注员对对话进行修正。
  1. 数据集规模（对话 -> 会话 -> 轮次）
  • 包含 50 段对话，每个对话都是两个智能体在聊天。
  • 时间跨度：一段 6-12 个月，平均 19 个会话，最多 35 个对话。一次会话指的是一个时间段的一次聊天。
  • 空间深度：一个对话平均 300 轮，平均 9k 的 token 数。

2. 三类评估任务
#

  1. 传统评估方法：关注 Agent 在当前轮的回复质量，如与参考答案的相似度，回答的一致性，共情能力等。
  1. 论文评估方法（任务 1、2 没有跨模态）

  • 1）问答任务——评估长期记忆能力

    • 单步推理（一轮对话问答）、多步推理、时序推理（时间推理）、常识推理（结合外部知识）、对抗推理（识别陷阱问题，抗幻觉和误导）。
    • 指标：F1 分数（精准匹配 + 部分匹配），RAG 模型额外加检索准确率。
      • LLM 的输出格式可能多样化，难以精确匹配，需确保答案关键词匹配，即部分匹配。
    • 测试对象：基础 LLMs、Long-context LLMs、RAG LLMs。
    • 检索器：DRAGON。

  • 2）总结任务——评估逻辑理解能力

    • 从对话历史总结出指定时间段内的符合事件图的核心事件，考察因果关系和时间顺序。
    • FactScore 指标：原子事实分解，强调事实描述的准确率与覆盖度，而非词汇相似度。
    • 测试对象：基础 LLMs、Long-context LLMs。

  • 3）生成任务——评估应用生成能力

    • 基于前两层，生成人设一致、叙事连贯的文本 + 图片响应。
    • 指标：自然语言生成指标，多模态对齐程度指标（模态之间的相关性和整体的对齐性）。
    • 测试对象：三种上下文对象配置——仅用原始对话、加全局摘要、加检索 observation 信息。

四、实验结果
#

1. 问答任务
#

  • 与人类基准相比，仍显落后。
  • Long Context-LLMs 在对抗性推理下，上下文越长，性能越低，幻觉太严重。
  • Long Context-LLMs 和 RAG-LLM 可以提升记忆能力，但推理能力有限。
  • 摘要具有高召回准确率，将其作为上下文并未显著提升性能，这可能是由于对话转换为摘要过程中的信息损失所致。

2. 摘要任务
#

  • ROUGE：词汇相似度，1 / 2 / L 表示连续几个词。
  • FactScore：事件相似度，准确率、召回率、F1 分数。
  • 长上下文大模型不一定优于基础大模型，可能是其并未有效利用上下文信息。
  • LLM 在该任务上可能犯的五类错误：
    • 漏信息（关键信息缺失）
    • 编信息（幻觉问题）
    • 误解信息（幽默、反讽等理解偏差）
    • 张冠李戴（归属错误）
    • 重要信息误判（把普通信息写入摘要）

3. 对话生成任务
#

五、应用启发
#

这篇论文对“长期交互型 AI 系统”的应用启发比较直接：

  • 长期记忆系统不能只靠长上下文：一次塞入更多历史，不等于真正记住。
  • RAG 的关键在记忆单元设计：如果 observation 粒度不好，检索再强也不稳定。
  • 适合做持续型个人助手 / 研究助理 / 陪伴型 Agent：重点不只是回答当前问题，而是跨多轮、多天、多主题后仍保持人设一致、事件连贯、记忆可追溯。
  • 评估要分层做：检索是否准、总结是否对、最终生成是否一致，这三层最好分开看。

从评测定位来看，这篇论文比较适合作为“长期记忆 Agent 评测基线”来看，而不只是看作一个普通对话数据集。

六、Limitations
#

  • 数据来源于 LLM 生成，即使有人工标注环节，但与真实世界对话仍有细微差别。
  • 多模态行为探索有限，图片完全可以由文字描述。
  • 数据集的生成仅针对英语开发。
  • 长文本自然语言生成评估。即便提示以短语简短作答，大语言模型仍倾向于生成冗长回答。
  • 单向情感风险：当前的 Agent 框架能够模拟真实生活对话，聊多了可能产生单向爱情（？？？）。
  • MLLM 生成的图像，可能导致错误信息的传播，存在安全问题。
  • 生成式智能体可能对人类协作产生不利影响。
